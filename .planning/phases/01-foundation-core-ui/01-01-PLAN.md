  <name>Task 1: Create Express server with types, seed data, and REST API</name>
  <files>
    server/package.json, server/tsconfig.json, server/src/index.ts,
    server/src/types.ts, server/src/routes/groups.ts, server/src/routes/analytics.ts,
    server/src/data/seed.ts, server/src/data/entries.json, server/src/data/groups.json,
    server/src/data/sessions/stub-session-01.json, server/src/data/sessions/stub-session-02.json
  </files>
  <action>
    **Step 1: Create server directory and initialize.**
    - `mkdir -p server/src/routes server/src/data/sessions`
    - Create server/package.json with name "ux-feedback-compiler-server", scripts: `"dev": "tsx watch src/index.ts"`, `"start": "tsx src/index.ts"`
    - Create server/tsconfig.json targeting ES2022, Node module resolution, strict mode, resolveJsonModule: true, esModuleInterop: true
    - `cd server && npm install express cors && npm install -D typescript @types/express @types/cors @types/node tsx`

    **Step 2: Create shared types in server/src/types.ts.**
    Use the exact type definitions from RESEARCH.md:
    - `FeedbackEntry` interface: id, text, source (union of "app-store"|"support-ticket"|"survey"|"slack"|"intercom"), submittedAt (ISO string), userName, screenshotUrl?, sessionId?, sentiment? ("positive"|"negative"|"neutral"), groupId
    - `FeedbackGroup` interface: id, title, summary, severity ("critical"|"high"|"medium"|"low"), entryCount, categories (string[]), aiSummary (object with problem, impact, reproductionSteps string[], suggestion), duplicateCount, trendDirection ("up"|"down"|"stable"), firstReported, lastReported
    - `AnalyticsData` interface: totalEntries, totalGroups, severityDistribution, categoryBreakdown, volumeByDate, topImpactedAreas

    **Step 3: Create seed data generator (server/src/data/seed.ts).**
    Build a script that generates the JSON files. The resulting data must have:
    - 8-12 feedback groups covering realistic UX issues: "Checkout Flow Crashes on Mobile", "Search Results Not Relevant", "Onboarding Tutorial Skips Steps", "Dashboard Loading Takes Too Long", "Payment Method Removal Fails", "Dark Mode Color Contrast Issues", "Export Feature Missing File Formats", "Notification Settings Not Persisting", "Profile Image Upload Errors", "Navigation Menu Accessibility Issues"
    - Each group has: a detailed aiSummary with problem, impact, reproductionSteps (3-5 steps), suggestion
    - Severity distribution: 2 critical, 3 high, 3 medium, 2-4 low
    - 200-300 feedback entries distributed across groups (15-40 per group, weighted by severity)
    - Entries from varied sources (app-store, support-ticket, survey, slack, intercom)
    - ~30% of entries have screenshotUrl referencing /screenshots/placeholder-NN.svg
    - ~15% of entries have sessionId referencing stub session files
    - Varied submittedAt dates over the past 30 days
    - Realistic vague/duplicate text: "it broke again", "checkout doesn't work", "same issue as before", "the thing with the payment is weird" alongside more detailed entries
    - Each group has 2-3 category tags from: navigation, performance, ux, payments, accessibility, data, mobile, desktop, onboarding, notifications, export

    Write the data directly into server/src/data/entries.json and server/src/data/groups.json as static JSON (not runtime generation -- avoids randomness between restarts).

    **Step 4: Create stub rrweb session files.**
    Create 2 minimal stub session JSON files in server/src/data/sessions/:
    - Each is a JSON array with 2 events: a Meta event (type: 4) and a FullSnapshot event (type: 2)
    - These establish the data structure for Phase 3. Content can be minimal placeholder data.

    **Step 5: Create Express server (server/src/index.ts).**
    - Import express, cors
    - Import group and analytics routers
    - Apply cors() and express.json() middleware
    - Mount routers: app.use('/api', groupsRouter), app.use('/api', analyticsRouter)
    - Listen on port 3001

    **Step 6: Create route handlers.**
    server/src/routes/groups.ts:
    - `GET /groups` -- returns all groups with entryCount computed from entries data. Map groups to include entryCount from entries where entry.groupId matches group.id
    - `GET /groups/:id` -- returns single group with its entries array. 404 if not found.
    - Load entries.json and groups.json via import (resolveJsonModule)

    server/src/routes/analytics.ts:
    - `GET /analytics` -- derives all analytics from seed data:
      - totalEntries: entries.length
      - totalGroups: groups.length
      - severityDistribution: count per severity level
      - categoryBreakdown: count per unique category across all groups
      - volumeByDate: count entries per day for past 30 days
      - topImpactedAreas: top 5 categories by group count
  </action>
