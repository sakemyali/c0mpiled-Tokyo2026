<task type="auto">
  <name>Task 2: Add entry submission endpoint and AI pipeline with processing endpoint</name>
  <files>server/src/routes/entries.ts, server/src/services/ai-pipeline.ts, server/src/routes/ai.ts, server/src/index.ts</files>
  <action>
1. Create `server/src/routes/entries.ts`:
   - POST /entries: accepts { text, source, userName } in body
   - Validate all three fields present, return 400 if missing
   - Create entry with id: `entry-${crypto.randomUUID().slice(0, 8)}`, submittedAt: new Date().toISOString(), groupId: "ungrouped"
   - Push to entries array from store, return 201 with entry object
   - GET /entries/ungrouped: return entries filtered by groupId === "ungrouped"

2. Create `server/src/services/ai-pipeline.ts`:
   - Import OpenAI from "openai" (uses OPENAI_API_KEY env var automatically)
   - `clusterEntries(entries: FeedbackEntry[])`: sends all entry texts as numbered list to gpt-4o-mini with json_object response_format. System prompt asks to group by underlying problem similarity into 5-15 groups, returning { groups: [{ title, entryIndices: number[], categories: string[] }] }. Temperature 0.3.
   - `summarizeGroup(title: string, entryTexts: string[])`: sends group's entry texts to gpt-4o-mini asking for { summary, severity, problem, impact, reproductionSteps, suggestion }. Temperature 0.3. Returns data matching FeedbackGroup shape.
   - `runPipeline(allEntries: FeedbackEntry[])`: Step 1 calls clusterEntries. Step 2 calls summarizeGroup in parallel (Promise.all) for each cluster. Builds FeedbackGroup[] with proper ids (group-ai-{index}), maps entry indices to groupIds. Returns { newGroups: FeedbackGroup[], assignments: Record<string, string> }. Wrap each step in try/catch, log progress to console.
   - Add `console.log` markers: "Step 1: Clustering N entries...", "Step 2: Summarizing N groups in parallel..."

3. Create `server/src/routes/ai.ts`:
   - Import entries, groups, isProcessing from store
   - POST /ai/process: Check isProcessing flag, return 409 if already running. Set isProcessing=true. Call runPipeline(entries). On success: clear groups array, push new groups, update each entry's groupId from assignments map. Set isProcessing=false. Return { success: true, groupCount }. On error: set isProcessing=false, return 500 with error message.
   - GET /ai/status: return { isProcessing } for client polling

4. Update `server/src/index.ts`:
   - Add imports for entries router and ai router
   - Mount: `app.use("/api", entriesRouter)` and `app.use("/api", aiRouter)`
   - Add startup check: if (!process.env.OPENAI_API_KEY) console.warn("WARNING: OPENAI_API_KEY not set -- AI endpoints will fail")
